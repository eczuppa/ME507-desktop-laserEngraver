Extracts a floating point value from a string. This function was taken directly from GRBL.

The following code is based loosely on the avr-libc strtod() function by Michael Stumpf and Dmitry Xmelkov and many freely available conversion method examples, but has been highly optimized for Grbl. For known CNC applications, the typical decimal value is expected to be in the range of E0 to E-4. Scientific notation is officially not supported by g-code, and the 'E' character may be a g-code word on some CNC systems. So, 'E' notation will not be recognized. NOTE: Thanks to Radu-Eosif Mihailescu for identifying the issues with using strtod().
