Detailed Description

File containing functions that interpret machine and G-code commands.

This file defines the functions necessary to run the full translation of commands coming from the serial port before sending them to the control task.

uint8_t interpret_machinecmd_line ( char *  line)

Function which interprets a line containing a machine command.

This function takes in a line containing a command for the laser that begins with a $, signalling that it is a machine command and not a line of gcode. It then interprets the command in the line and returns on the information. Disclaimer: This function currently only supports homing commands. More will be added in the future.

void task_translate ( void *  p_params)

Task which reads data from the serial port, translates it, and sends it where it needs to go.

This task function reads a line from the read_chars queue of gcode or other commands and splits it up into the separate commands. Commands are then sent to the control task via queues and shares. Gcode is translated from X Y and F values into desired positions and feedrates for both motors; _A_setpoint, _A_feed, _B_setpoint and _B_feed, which are class member data of the class Kinematics_coreXY. Regular machine commands such as homing are also translated to motor commands, which are then sent to the control task so it may run them.
